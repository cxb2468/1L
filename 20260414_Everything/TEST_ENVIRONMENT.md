# 测试环境配置和测试方法

## 1. 测试环境配置

### 1.1 硬件环境
- **操作系统**: Windows 10/11 x64
- **CPU**: 至少双核处理器
- **内存**: 至少4GB RAM
- **磁盘空间**: 至少100MB可用空间

### 1.2 软件环境
- **Python**: 3.7或更高版本
- **依赖项**: 
  - psutil (用于进程管理)
  - pywin32 (用于Windows API调用)

### 1.3 SDK文件配置
由于无法直接从官网下载Everything 1.5a压缩包，我们使用了以下替代方案：

1. **使用ref_materials目录中的SDK文件**:
   - `Everything64.dll` (版本信息: 大小97800字节, 修改时间2023/06/13 13:23:32)
   - 该文件位于 `d:\Trae_projects\EverythingHelper\ref_materials\Everything64.dll`

2. **SDK文件搜索路径**:
   - 程序所在目录
   - ref_materials目录
   - Everything安装目录
   - 系统默认路径 (C:/Program Files/Everything/)

## 2. 测试方法

### 2.1 功能测试

#### 2.1.1 版本检测测试
- **测试目标**: 验证系统是否能正确检测Everything版本
- **测试步骤**:
  1. 运行 `test_version_detection.py` 脚本
  2. 检查输出结果是否正确显示版本信息

#### 2.1.2 SDK适配测试
- **测试目标**: 验证SDK适配器是否能正确加载和使用不同版本的SDK
- **测试步骤**:
  1. 运行 `test_sdk_adapter.py` 脚本
  2. 检查SDK加载状态和搜索功能是否正常

#### 2.1.3 运行状态管理测试
- **测试目标**: 验证是否能正确检测和管理Everything运行状态
- **测试步骤**:
  1. 运行 `test_status_management.py` 脚本
  2. 检查运行状态检测、启动功能是否正常

#### 2.1.4 界面功能测试
- **测试目标**: 验证用户界面功能是否正常
- **测试步骤**:
  1. 运行 `main.py` 启动应用
  2. 测试工具栏按钮点击功能
  3. 测试搜索功能
  4. 测试结果显示和交互功能

### 2.2 兼容性测试

#### 2.2.1 版本兼容性测试
- **测试目标**: 验证应用是否能兼容Everything 1.4和1.5a版本
- **测试步骤**:
  1. 在安装了Everything 1.4的环境中运行应用
  2. 在安装了Everything 1.5a的环境中运行应用
  3. 检查两种环境下应用是否正常工作

#### 2.2.2 系统兼容性测试
- **测试目标**: 验证应用在不同Windows版本上的兼容性
- **测试步骤**:
  1. 在Windows 10上运行应用
  2. 在Windows 11上运行应用
  3. 检查应用是否正常工作

### 2.3 性能测试

#### 2.3.1 搜索性能测试
- **测试目标**: 验证搜索功能的性能
- **测试步骤**:
  1. 执行不同复杂度的搜索查询
  2. 记录搜索响应时间
  3. 检查结果准确性

## 5. 测试结果分析

### 5.1 预期结果
- **版本检测**: 能正确检测系统中安装的Everything版本

   - 确保SDK文件与系统架构匹配 (32位/64位)

2. **Everything未运行**:
   - 检查Everything是否已安装
   - 手动启动Everything

3. **搜索无结果**:
   - 检查Everything索引是否完成
   - 检查搜索查询是否正确
